# MCP IDE Link - 機能一覧 (Available Tools)

MCP IDE Link 拡張機能は、AIエージェント（Cline, Cursor等）がVS Codeのネイティブ機能を直接利用できるようにするためのMCPサーバーです。
現在、以下の9つの強力なツール（Tools）を提供しています。

---

## 状態取得・コンテキスト理解ツール
AIが「ユーザーが現在どこを見て作業しているか」を正確に把握するための機能です。

### 1. `get_active_editors`
- **概要**: ユーザーが現在開いているすべてのタブ（エディタ）と、一番手前でアクティブになっているファイルの絶対パスを取得します。
- **入力引数**: なし
- **用途**: AIがコンテキストとしてどのファイルを読み込むべきか判断する際に使用します。

### 2. `get_cursor_position`
- **概要**: 現在アクティブなエディタのファイルパス、カーソル位置（行・列）、および「選択（ハイライト）されているテキスト」を取得します。
- **入力引数**: なし
- **用途**: ユーザーから「ここのコードを直して」と指示された際に、AIが「ここ」がどこなのかを正確に特定するために使用します。

---

## ファイル編集・状態管理ツール
AI自身がファイルを安全に編集し、その状態を管理するための機能です。

### 3. `apply_workspace_edit`
- **概要**: ファイル内の特定の文字列を、安全な方法（未保存状態のタブ操作）で置換します。
- **入力引数**:
  - `filePath`: 対象ファイルの絶対パス
  - `target`: 置換対象となる元の文字列（完全一致）
  - `replacement`: 新しく置換する文字列
- **用途**: CLIでのファイル上書きとは異なり、VS Codeの「Undo（Ctrl+Z）」が効く未保存状態で編集が行われるため、人間が変更内容を目視で確認してから保存できます。

### 4. `save_all_dirty_files`
- **概要**: 編集によって「未保存状態（Dirty）」になっているすべてのワークスペースのファイルを一括で保存します。
- **入力引数**: なし
- **用途**: `apply_workspace_edit` で編集した複数のファイルを、AIが自信を持ってプログラム的に保存（確定）させたい場合に使用します。

### 5. `close_all_editors`
- **概要**: VS Code上で開いているすべてのタブ（エディタ）を閉じます。
- **入力引数**: なし
- **用途**: 調査や編集のためにAIが大量に開いたファイルを一掃し、人間の作業スペースをクリーンアップします。

## LSP (Language Server Protocol) 連携ツール
単なるテキスト検索では不可能な、IDEならではの高度なコード解析機能です。

### 6. `rename_symbol`
- **概要**: 言語サーバー（LSP）を利用して、変数名や関数名をプロジェクト全体で安全にリネーム（一括置換）します。
- **入力引数**:
  - `filePath`: 対象シンボルが存在するファイルの絶対パス
  - `line`: シンボルのある行番号（0始まり）
  - `character`: シンボルのある文字オフセット（0始まり）
  - `newName`: 新しい名前
- **用途**: 通常のテキスト置換では関係ない同名文字列まで壊してしまうリスクがありますが、これを使うと参照関係を正しく理解した上で安全にリファクタリングできます。

### 7. `find_references`
- **概要**: 指定したシンボル（変数や関数）が、コードベースの「どこから呼び出されているか・参照されているか」をすべて取得します。
- **入力引数**:
  - `filePath`: 対象ファイルの絶対パス
  - `line`: 行番号（0始まり）
  - `character`: 文字オフセット（0始まり）
- **用途**: AIがコードを変更する前に「この関数を変えると、どこに影響が出るか」を正確に調査できます。

### 8. `go_to_definition`
- **概要**: 指定した関数や変数が「もともとどこで定義・作成されたものか」の参照先（ファイルパスと行数）を取得します。
- **入力引数**:
  - `filePath`: 呼び出し元のファイルの絶対パス
  - `line`: 行番号（0始まり）
  - `character`: 文字オフセット（0始まり）
- **用途**: AIが見知らぬ関数を見つけた際に、その関数の実装や型定義をピンポイントで調べに行くことができます。

### 9. `get_diagnostics`
- **概要**: ファイル内で発生している静的解析のエラーや警告（エディタ上で赤い波線・黄色い波線で表示されるもの）のリストを取得します。
- **入力引数**:
  - `filePath` (省略可): エラーを取得したいファイルの絶対パス。省略した場合はワークスペース全体のエラーを取得します。
- **用途**: コードを編集した直後にこれを利用して、文法エラーや型エラー（TypeScriptのコンパイルエラーなど）をユーザーの代わりにAI自身が発見し、自動修正ループを回すことができます。
