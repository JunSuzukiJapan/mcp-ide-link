# MCP IDE Link - 将来の開発アイデア

AIエージェント（Cline等）のコーディング能力を最大化するために、通常のCLIツール（grep, sed等）では実現できない「VS Code独自のLSP機能や状態情報」を提供するツールのアイデア一覧です。

これらの機能をMCPサーバー経由でAIに提供することで、AIの状況把握力や安全なファイル操作能力が飛躍的に向上します。

## 優先度 高：AIの「コンテキスト理解」を助ける機能

エディタの現在の状態をAIに共有することで、「AIがこれから何をすべきか」をユーザーの視線と一致させる機能です。

### 1. `get_active_editors` (現在開いているタブ・アクティブなファイルの取得)
- **目的**: ユーザーが現在 VS Code で見ているファイル群をAIに知らせる。
- **効果**: AIはゼロからプロジェクト全体を捜索するのではなく、「ユーザーが今開いているファイルの中に問題があるはずだ」と即座に当たりをつけることができる。

### 2. `get_cursor_position_and_selection` (カーソル位置・選択範囲のテキスト取得)
- **目的**: ユーザーが選択（ハイライト）しているコードの正確な範囲と文字列をAIに提供する。
- **効果**: 「ここをおかしい」「この関数をリファクタして」といったチャット指示に対して、AIが対象のコードを一瞬で正確に特定し、ミスコミュニケーションをなくす。

## 優先度 中：AIの「安全な操作とクリーンアップ」を助ける機能

AIが自律して行動する際のファイル管理を助ける機能です。

### 3. `save_all_dirty_files` (未保存ファイルの保存)
- **目的**: `apply_workspace_edit` 等で自動変更されて「未保存（Dirty）」になっているファイルを一括で保存する。
- **効果**: ユーザーがVS Code上で差分を確認し、「よし、これでOK」とAIに伝えたタイミングで、AIにすべての保存処理を委任できる。

### 4. `close_tabs` (開いたタブの整理)
- **目的**: AIが調査のために大量に開いてしまった邪魔なファイルを閉じる。
- **効果**: 作業環境のクリーンアップもAIに任せることができる。

## 優先度 中～長期：IDEネイティブ（言語サーバー連携）の高度な機能

通常のシェル操作（findやgrep）では不可能な、IDEの「言語解析能力」をAIに提供します。

### 5. `find_references` (すべての参照元の検索)
- **目的**: 特定の関数や変数が「コードのどこで使われているか」をLSPを使って厳密に検索する。
- **効果**: 関数名や引数を変更する大規模なリファクタリング時に、AIが影響範囲を完璧に把握し、破壊的変更によるバグを防ぐことができる。

### 6. `go_to_definition` (定義元へのジャンプ / パス取得)
- **目的**: AIが未知の関数呼び出しを見つけたとき、それがどこで定義されているかをLSPに尋ねてファイルの該当行を取得する。
- **効果**: AIが自力でコードリーディング（関連ファイルの追跡）を行う時間を大幅に短縮し、より正確な修正案を提示できるようになる。

### 7. `get_diagnostics` (リアルタイムのエラー・警告一覧の取得)
- **目的**: VS Codeの「問題」パネルに表示される赤い波線（Linterのエラー、TypeScriptの型エラーなど）の情報をすべて取得する。
- **効果**: AIがファイルを修正した直後にこのツールを呼ぶことで、ターミナルでビルドコマンドを走らせなくても、AI自身で修正内容の妥当性（コンパイルエラーや型エラーが起きていないか）を検証できるようになる。
